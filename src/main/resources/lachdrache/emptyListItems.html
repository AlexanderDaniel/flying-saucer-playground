List items with list-style: none are used in this example. With li:before and the content property a custom list item marker is added.

Additionally li:before uses position: absolute. For that to work the enclosing li element has to use position: relative. This is very well explained in Learn CSS Layout. That has following effects:

Empty list item at the beginning is not displayed:

  • two
  • three
  • four
  • five

Empty list item in the middle is not displayed:

  • one
  • two
  • four
  • five
  • one
  • three
  • five

But an empty list item at the end gets displayed:

  • one
  • two
  • three
  • four

Do you have an idea why?

I have an idea. The list item markers are drawn above each other. So let's try with a ordered list:

Empty list item at the beginning:

  2. two
  3. three
  4. four
  5. five

Empty list item in the middle:

  1. one
  2. two
  4. four
  5. five
  1. one
  3. three
  5. five

Empty list item at the end:

  1. one
  2. two
  3. three
  4. four

It has other side effects as well. The spacing between the list and the next paragraph is not correct.
